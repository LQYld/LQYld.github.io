# oneOf 处理

在 webpack.config.js 文件中 module.rules 里面使用的 loader 在编译的过程中，每一个文件都会根据 test 的正则表达式 从第一个 loader 一直检验到最后一个。会存在不必要的检测也会被检测一次，非常影响性能效率。所以 webpack提供了 oneOf 进行处理优化了性能。

# 使用 oneOf 进行操作

module:{
    rules:[
        {   // 以下 loader 只会匹配一个
            // 注意：不能有两个配置处理同一种类型文件
            oneOf:[
                {test:/\.css$/,use:['style-loader','css-loader']},
                {test:/\.less$/,use:['style-loader','css-loader','less-loader']},
            ]          
        }    
]
}

注意

oneOf 中 只会匹配执行一种 loader。
但是如果同时使用了 eslint-loader 及 babel-loader，需要将先执行的 loader 从 oneOf 中单独提出。

例

module:{
    rules:[
        {test:/\.js$/,use:['eslint-loader']},
        {   // 以下 loader 只会匹配一个
            // 注意：不能有两个配置处理同一种类型文件
            oneOf:[
                {test:/\.css$/,use:['style-loader','css-loader']},
                {test:/\.less$/,use:['style-loader','css-loader','less-loader']},
                {test:/\.js$/,use:['babel-loader']},
            ]          
        }    
]
}
