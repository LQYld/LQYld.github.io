# 生命周期

解析

  • 先执行 Nuxt 的生命周期。
  • 再执行 Vue 的生命周期。 beforeCreated && created
  • 最后执行其他的生命周期。

# nuxtServerInit （服务器初始化）

解析

nuxtServerInit 函数 会在服务器中运行，会的到 Vuex 的状态树，及服务端上下文信息。
服务端上下文信息 有 该项目的配置项信息，及请求、请求头等等信息。

    nuxtServerInit(store,context){
        console.log('nuxtServerInit',store,context)
    }

推荐

nuxtServerInit 函数的使用场景，一般利用于，服务器初始化绑定初始值。

# middleware （中间件运行）

# middleware 配置 在 nuxt.config.js 中

我们要使用中间件运行时

    1. 需要在 nuxt.config.js 文件中进行配置。
    1. 在 nuxt.config.js 文件中找到字段名为 router 的配置项。
    1. 在 router 中定义 middleware 字段名 设置中间件函数。 如例：例 middleware-1。
    1. 在目录结构中的 middleware 文件夹中 编写 中间件函数。 如例：例 middleware-2。

# 例 middleware-1。

module.exports = {
  router: {
    middleware: "auth",
  },
};

# 例 middleware-2。

 cd middleware
 mkdir auth.js

# 编写 auth.js 文件

注意

middleware 可以作为 项目路由的 全局守望。

export default (context) => {
  // context 服务端上下 文信息
  console.log(middleware);
};

解析 （context）

context 服务端上下 文信息，可以获取到 store 状态树、路由、请求头、请求参数等一系列信息。
context 为对象格式。

# middleware 定义在 layout 中

# default.vue 文件中

export default {
  middleware: "auth",
};

扩展

middleware 及可以定义为外部函数，也可以定义为内部函数。
如 例：middleware-3。

# 例 middleware-3。

export default {
  middleware(context) {
    console.log("middleware");
  },
};

# 单页面中使用 middleware

与之前两种方式相同。在需要使用的页面中使用即可。

export default {
  // 外部方式引用
  middleware: "auth",
  // 内部函数方式
  middleware(context) {
    console.log("middleware");
  },
};

# validate() 校验参数

警告

validate() 用于判断是否允许进入该页面。
validate() 最好不要全局使用。

// 返回 布尔值 表示是否允许进入该页面
validate({params,query}){
    console.log('validate');
    return Boolean;
}

# asyncData & fetch 异步函数

注意

asyncData 与 fetch 执行没有先后顺序。

// 页面中使用
export default {
  asyncData() {},
  fetch() {},
};

扩展（asyncData & fetch）的区别

asyncData 异步获取后，将数据注入页面中的 data 中。 返回的是对象。 fetch 异步获取后，将数据注入项目中的 Vuex 状态树中。 并无返回，直接进行 store 操作。

# 关于 客户端生命周期

客户端的生命周期与原生 Vue 的相同。

注意

服务端渲染 不支持 原有 Vue 中 keep-alive 组件。
所以不存在 activated() 及 deactivated()。

# 关于 window 对象

警告

在服务端的生命周期内，是获取不到 window 对象的。

# 不同生命周期内 this 的指向

# 服务端生命周期内

this 的指向是 undefined。可以使用 服务端上下文 context。

# 客户端生命周期内

this 的指向时 当前页面 或 当前组件本身，Vue。
